## 二、各问题修复方案

### 修复1 & 2 & 3：jingdian.html — 替换全部 API 调用为本地数据

**根本原因**：三个 `async` 方法通过 `fetch()` 请求本地未运行的后端服务，导致 `net::ERR_CONNECTION_REFUSED`，`loading` 状态永不结束，页面空白。

**修复方案**：
- 新增 `getLocalSpotDatabase()` 方法，内嵌包含全部27个景点的完整数据对象（id、title、subtitle、cover、tags、content）。数据与 `index.html` 中 `qinghaiPlaces`、`gannanPlaces`、`chuanxiPlaces`、`northernTibetPlaces` 四个区域的景点完全对应。
- `loadSpotDetail()`：从本地数据库按 URL `id` 参数同步查找，不再使用 `fetch`，无网络依赖。
- `loadCommentList()`：从 `localStorage` 读取 key 为 `spot_comments_${spotId}` 的 JSON 数组。
- `submitComment()`：将新评论写入 `localStorage`，包含 id（时间戳）、content、userName、createTime 字段，写完后重新调用 `loadCommentList()` 刷新列表。

**修改位置**：`jingdian.html` 第680-764行（script块内 methods 区域）

---

### 修复4 & 5 & 6：minsu.html — 替换全部 API 调用为本地数据

**根本原因**：同 jingdian.html，三个 `async` 方法请求不存在的后端。

**修复方案**：
- 新增 `getLocalFolkDatabase()` 方法，内嵌10条安多民俗数据（藏历新年、赛马节、唐卡艺术、锅庄舞、婚俗、酥油花、格萨尔史诗、藏族服饰、青稞酒文化、转经习俗）。
- `loadFolkDetail()`：从本地数据库同步查找。
- `loadCommentList()`：从 `localStorage` 读取 key 为 `folk_comments_${folkId}` 的 JSON 数组。
- `submitComment()`：将新评论写入 `localStorage`，包含完整字段，写完后刷新列表。

**修改位置**：`minsu.html` 第698-782行（script块内 methods 区域）

---

### 修复7：index.html — 川西北导航链接从 `href="#"` 改为实际路径

**根本原因**：川西北安多片区8个子菜单条目均使用 `href="#"`，点击后页面滚动到顶部，且 `currentScenic` 全部误设为 `'songpan'`。

**修复方案**：
- 8个条目均改为实际 HTML 文件路径，与 `chuanxiPlaces` 数据中的 `link` 字段完全一致：
  - 阿坝州 → `aba/aba.html`
  - 若尔盖草原 → `ruoergai/ruoergai.html`
  - 松潘古城 → `songpan/songpan.html`
  - 年宝玉则 → `nianbaoyuze/nianbaoyuze.html`
  - 马尔康卓克基官寨 → `zhuokeji/zhuokeji.html`
  - 九曲黄河第一湾 → `huanghewan/huanghewan.html`
  - 红原大草原 → `hongyuan/hongyuan.html`
  - 阿坝县各莫寺 → `gemoshi/gemoshi.html`
- 同时修正各条目的 `currentScenic` 标识符，使激活状态正确。

**修改位置**：`index.html` 第1519-1526行

### 修复8：图片双扩展名 `mq1.jpg.jpg`

**根本原因**：上传文件时文件名误带双扩展名，`image1/` 目录中实际文件名为 `mq1.jpg.jpg`，但被 HTML 引用时应为 `mq1.jpg`。

**修复方案**：
1. `index.html` 第2029行：将 `image1/mq1.jpg.jpg` 改为 `image1/mq1.jpg`。
2. 同时在 `image1/` 目录中复制文件：`mq1.jpg.jpg` → `mq1.jpg`，使新路径有效。

---

### 修复9：地图容器空白提示

**根本原因**：高德地图通过动态注入 `<script>` 加载，网络不通或 API Key 失效时 `#amap-container` 显示纯空白。

**修复方案**：在 `#amap-container` 内添加 HTML 占位内容（图标 + 提示文字），高德地图加载成功后会自动覆盖此占位内容；加载失败时用户看到友好提示而非空白。

**修改位置**：`index.html` 第1667行

## 四、后端建议

当前项目完全前端静态化，如未来需要真实的用户系统、数据持久化（跨设备同步评论、收藏等），建议独立建立后端服务。

### 推荐方案：Java Spring Boot

```
backend/
├── pom.xml
├── src/main/java/com/tibetan/
│   ├── controller/
│   │   ├── TourSpotController.java    # GET /api/tour-spot/detail/{id}
│   │   ├── FolkController.java        # GET /api/folk/detail/{id}
│   │   └── CommentController.java     # GET/POST /api/comment/list, /api/comment/add
│   ├── service/
│   ├── repository/
│   └── model/
│       ├── TourSpot.java
│       ├── Folk.java
│       └── Comment.java
└── src/main/resources/
    └── application.yml                # server.port=8080, 配置MySQL/H2
```

**最小API接口列表**：

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/tour-spot/detail/{id}` | 景点详情 |
| GET | `/api/tour-spot/comment/list/{spotId}` | 景点评论列表 |
| POST | `/api/tour-spot/comment/add` | 提交景点评论 |
| GET | `/api/folk/detail/{id}` | 民俗详情 |
| GET | `/api/folk/comment/list/{folkId}` | 民俗评论列表 |
| POST | `/api/folk/comment/add` | 提交民俗评论 |

响应格式统一为：
```json
{ "code": 200, "msg": "success", "data": { ... } }
```

前端已按此格式编写（原API调用代码中的 `data.code === 200` 判断），接入后端后只需将 `jingdian.html` 和 `minsu.html` 的本地数据方法切换回 `fetch` 调用即可。

### 备选方案：Node.js + Express

```
backend/
├── package.json
├── server.js
├── routes/
│   ├── tourSpot.js
│   ├── folk.js
│   └── comment.js
└── data/
    ├── spots.json      # 可直接复制 getLocalSpotDatabase() 的数据
    └── folks.json      # 可直接复制 getLocalFolkDatabase() 的数据
```

Node.js 方案启动更快，适合快速原型验证。数据库可先用 JSON 文件，后期迁移到 MySQL 或 MongoDB。

## 五、第三轮修复记录（2026-04-20）

### 修复10：个人中心 / 收藏 / 浏览历史功能实现

**问题描述**：用户下拉菜单中的"个人中心"、"我的收藏"、"浏览历史"三项功能仅有入口，点击后无实际功能响应。

**修复方案**：
- 实现侧滑个人中心面板，展示用户名、收藏统计、浏览足迹
- 收藏功能：以 `user_favorites` 为 Key，JSON 数组存储收藏的景点/民俗 ID，支持添加与取消收藏
- 浏览历史：以 `browse_history` 为 Key，JSON 数组存储浏览记录（页面标题、URL、时间戳），FIFO 策略最多保留 50 条
- 个人中心面板内展示收藏数量统计和最近浏览时间线

---

### 修复11：地图四区域均分布局

**问题描述**：地图右侧地名列表四个区域高度不一致，青海区域占据过多空间，藏北区域被挤压至底部需要滚动。

**修复方案**：
- 四大区域改为折叠式交互（Accordion），默认全部折叠，点击区域标题展开/收起
- 使用 Flexbox `flex: 1` 均分右侧面板空间，四区域块等高排列
- 区域之间添加统一边线间隔（`border-bottom: 1px solid #eee`，间距 `8px`）
- 标记点点击时自动展开对应区域、收起其他区域

---

### 修复12：藏文笔记侧边栏多条管理

**问题描述**：藏文笔记页面仅支持单条笔记编辑，无法管理多条笔记，用户体验受限。

**修复方案**：
- 新增左侧侧边栏笔记列表，展示所有已保存笔记的标题和更新时间
- 支持新建、切换、删除笔记操作
- 数据结构从单条字符串升级为 JSON 数组：`[{ id, title, content, createTime, updateTime }]`
- localStorage Key 统一为 `tibetan_notes`

---

### 修复13：那曲地区 7 页风格统一

**问题描述**：`naqu/` 目录下 7 个景点页面（naqu、namucuo、kekexili、selincuo、tanggulashankou、zandansi、kulouqiang）使用旧版深色主题，与全站亮色米白风格不一致。

**修复方案**：
- 全部 7 页统一为亮色米白主题（`background: #FAF9F7`）
- 统一色彩变量：藏红 `#9B2B25` + 金色 `#D4AF37`
- 统一页面结构：top-bar → header → banner(500px) → main-container → sections → footer
- 统一组件样式：highlight-box、card-grid、feature-list、tips-box

---

### 修复14：全站标题 / 导航 / viewport 统一

**问题描述**：部分页面 `<title>` 标签未设置或与内容不符，多个景点页面缺少 `<meta name="viewport">` 标签导致移动端显示异常，导航栏样式在各页面间存在差异。

**修复方案**：
- 全站所有 HTML 文件统一添加 `<meta name="viewport" content="width=device-width, initial-scale=1.0">`
- `<title>` 统一格式为："景点名称 - 雪域安多 · 藏韵传承"
- 导航栏结构和样式统一，确保各页面间跳转体验一致
- 全站返回顶部按钮统一样式和行为

---

### 修复15：登出 API 调用修复

**问题描述**：登出功能调用 `fetch('/api/user/logout')`，由于无后端服务，请求失败后控制台报错，且登出流程未正确执行。

**修复方案**：
- 移除对 `/api/user/logout` 的 `fetch` 调用
- 登出操作改为直接清除 localStorage 中的 `authToken`、`userName` 等认证信息
- 清除后刷新页面状态，恢复未登录 UI
