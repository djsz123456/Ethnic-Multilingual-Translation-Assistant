# 注册功能说明文档

## 1. 需求分析

### 1.1 前端交互需求
- **用户认证流程**：用户需要通过登录和注册功能访问平台的核心功能
- **登录验证**：用户点击平台任何功能按钮时，如未登录则自动弹出登录/注册页面
- **注册流程**：新用户需要通过手机号注册，获取验证码并设置密码
- **登录状态管理**：登录后显示用户信息，提供个人中心、收藏、历史记录等功能
- **用户体验**：响应式设计，适配不同设备屏幕尺寸，提供流畅的动画效果

### 1.2 页面需求
- **登录/注册弹窗**：包含登录和注册两个标签页，支持平滑切换
- **表单验证**：实时表单验证，提供明确的错误提示
- **用户按钮**：右上角浮动用户按钮，根据登录状态显示不同状态
- **下拉菜单**：登录后显示用户下拉菜单，包含个人中心、收藏、历史记录等选项
- **Toast提示**：操作结果的实时反馈，包括成功、错误、信息提示

### 2.2 核心功能流程

#### 注册流程
1. 用户点击右上角用户按钮，弹出登录/注册弹窗
2. 切换到注册标签页
3. 输入用户名、手机号，点击获取验证码
4. 输入收到的验证码和设置密码
5. 点击注册按钮，提交注册信息
6. 注册成功后自动切换到登录标签页

#### 登录流程
1. 用户点击右上角用户按钮，弹出登录/注册弹窗
2. 在登录标签页输入用户名/手机号和密码
3. 点击登录按钮，提交登录信息
4. 登录成功后关闭弹窗，显示用户登录状态

#### 功能访问控制流程
1. 用户点击平台任何功能按钮
2. 系统检查登录状态
3. 未登录：自动弹出登录/注册弹窗
4. 已登录：正常执行功能操作

## 5. 实现细节

### 5.1 表单验证
- **用户名验证**：非空，至少2个字符
- **手机号验证**：符合中国大陆手机号格式
- **验证码验证**：非空，至少4位
- **密码验证**：6-20位
- **确认密码验证**：与密码一致

### 5.2 状态管理
- 使用localStorage存储登录状态：`authToken`、`userId`、`userName`
- 页面加载时自动检查登录状态
- 登录状态过期时自动清除本地存储

### 5.3 用户体验优化
- **动画效果**：弹窗显示/隐藏动画、按钮悬停效果、Toast提示动画
- **实时反馈**：表单验证实时反馈、操作结果Toast提示
- **响应式设计**：适配PC、平板、手机等不同设备
- **加载状态**：登录/注册按钮加载状态显示

### 5.4 安全性
- 使用HTTPS协议进行API通信
- 密码使用加密传输
- JWT令牌认证机制
- 验证码防止恶意注册
