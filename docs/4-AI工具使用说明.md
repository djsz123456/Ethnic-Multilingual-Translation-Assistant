## 总体说明

### AI工具使用概况

本作品在开发过程中使用了 **Claude Code**（Anthropic公司的AI编程助手CLI工具）辅助完成部分前端代码开发工作。开发过程中先后使用了claude-sonnet-4-6和claude-opus-4-7两个模型版本。AI工具主要用于以下环节：

1. **页面模板生成**：基于已有设计规范（guoluo.html参考页），批量生成统一风格的景点专题页面HTML/CSS框架
2. **API集成开发**：藏汉互译功能的MyMemory API对接代码
3. **交互优化**：卡片全区域点击的CSS方案设计
4. **功能增强**：个人中心侧滑面板、收藏/浏览历史、笔记多条管理等功能开发
5. **全站审查**：页面风格统一（暗色→亮色主题迁移）、导航/标题/viewport一致性修复
6. **代码清理**：冗余代码的定位与移除

### 人工工作占比

| 工作内容 | 人工占比 | AI辅助占比 |
|---------|---------|-----------|
| 作品创意与选题 | 100% | 0% |
| 整体架构设计 | 100% | 0% |
| 视觉设计语言定义（CSS Design System） | 90% | 10% |
| 文化内容编撰与核实（35+景点） | 85% | 15% |
| 前端页面代码（HTML/CSS/JS） | 40% | 60% |
| API功能集成（翻译/地图） | 30% | 70% |
| 用户系统开发（认证/收藏/历史） | 45% | 55% |
| 全站风格统一与UI修复 | 50% | 50% |
| 测试与修复 | 70% | 30% |
| 文档撰写 | 60% | 40% |

**综合评估**：AI工具在本项目中主要承担了重复性较高的HTML/CSS模板代码生成和功能模块的框架搭建工作。所有文化内容、创意设计、架构决策和质量把控均由团队成员人工完成。AI生成的代码均经过人工审查、功能测试和必要的修改后方才采用。特别是35+景点的文化内容、藏文相关数据和地理信息均由人工调研编撰，确保学术准确性和文化敏感性。
