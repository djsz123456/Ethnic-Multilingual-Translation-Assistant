## 第一章 需求分析

### 1.1 项目背景与目标

安多藏区是中国三大藏区之一，横跨青海、甘肃甘南、四川阿坝及西藏那曲，拥有深厚的藏传佛教文化、游牧文化和民俗传统。然而，当前互联网上缺乏体系化、高质量的安多藏族文化展示平台，信息分散、内容碎片化，不利于文化的传播与保护。

本平台旨在构建一个**综合性安多藏族文化传播Web平台**，系统展示安多藏区的自然风光、历史人文、宗教文化与民俗传统，同时提供实用的藏文学习工具与交互式地图导航，推动安多藏族文化的数字化保护与传播。

### 1.2 面向用户

- 对藏族文化感兴趣的普通公众
- 计划前往安多藏区旅游的旅行者
- 藏语学习者和研究人员
- 文化保护与传承工作者
- 高校民族学、人类学相关专业师生

### 1.3 主要功能

| 功能模块 | 说明 |
|---------|------|
| 藏地旅游展示 | 覆盖青海、甘南、川西阿坝、藏北那曲四大区域共35+景点文化专题页，含自然风貌、历史人文、文化意义、旅游服务 |
| 交互式地图导航 | 基于高德地图JS API的四区域折叠式景点列表，支持标记点击自动定位、信息窗口弹出、区域分组导航 |
| 安多民俗介绍 | 展示安多藏族传统服饰、饮食、节庆、游牧生活等民俗文化 |
| 藏汉互译工具 | 基于MyMemory API的藏汉双向即时翻译，支持Ctrl+Enter快捷翻译、AbortSignal超时控制 |

| 藏文键盘 | 虚拟藏文输入法键盘，支持藏文字符直接输入 |
| 藏族音乐 | 安多藏族传统音乐在线播放与欣赏 |

**架构说明**：

- **用户层**：支持桌面端与移动端浏览器，通过CSS媒体查询实现响应式自适应
- **前端展示层**：纯静态HTML架构，Vue 3负责首页及工具页的响应式数据绑定，自研CSS Design System提供统一视觉规范，Vanilla JS模块处理各景点页面的独立交互逻辑
- **数据持久化**：采用Web Storage API（localStorage），以JSON序列化方式存储用户认证信息、收藏数据、浏览历史和笔记内容
- **外部服务层**：通过HTTPS协议调用高德地图JS API（地图渲染与标记管理）、MyMemory API（藏汉翻译）、Google Fonts（字体加载）、jsDelivr CDN（Vue和Font Awesome库分发）

#### 3.2.2 高德地图 JS API 交互式导航

地图模块采用高德地图JS API v1.4.15，实现动态脚本加载、标记管理、信息窗口与四区域分组导航：

```javascript
// 动态脚本加载 — 延迟初始化策略
function loadAMapScript() {
  return new Promise((resolve, reject) => {
    const script = document.createElement('script');
    script.src = 'https://webapi.amap.com/maps?v=1.4.15&key=YOUR_KEY';
    script.onload = resolve;
    script.onerror = reject;
    document.head.appendChild(script);
  });
}

// 四区域景点标记管理
```

#### 3.2.3 藏汉互译 — MyMemory API 集成

翻译功能采用 MyMemory Translation API，通过RESTful接口实现免费、无需后端代理的浏览器端即时翻译，配合AbortSignal实现超时控制：

```javascript
async function doTranslate() {
  const langpair = srcLang === 'zh' ? 'zh-CN|bo' : 'bo|zh-CN';
  const url = `https://api.mymemory.translated.net/get`
            + `?q=${encodeURIComponent(q)}&langpair=${langpair}`;
  
  // AbortSignal超时控制 — 10秒自动取消
  const resp = await fetch(url, { signal: AbortSignal.timeout(10000) });
  const data = await resp.json();
  
  if (data.responseStatus === 200) {
    outputText.value = data.responseData.translatedText;
  } else {
    showError('翻译服务暂时不可用，请稍后重试');
  }
}

// Ctrl+Enter 快捷翻译
document.addEventListener('keydown', (e) => {
  if (e.ctrlKey && e.key === 'Enter') doTranslate();
});
```

#### 3.2.4 用户认证模块（localStorage方案）

采用Web Storage API实现轻量级用户认证系统，无需后端服务器：

```javascript
// 用户注册 — JSON序列化存储
function registerUser(username, password) {
  const users = JSON.parse(localStorage.getItem('users') || '[]');
  if (users.find(u => u.username === username)) {
    return { success: false, message: '用户名已存在' };
  }
  users.push({ username, password, createdAt: Date.now() });
  localStorage.setItem('users', JSON.stringify(users));
  return { success: true };
}

// 登录状态管理
function login(username, password) {
  const users = JSON.parse(localStorage.getItem('users') || '[]');
  const user = users.find(u => u.username === username && u.password === password);
  if (user) {
    localStorage.setItem('currentUser', JSON.stringify(user));
    return { success: true };
  }
  return { success: false, message: '用户名或密码错误' };
}

// 全局登录状态检测
function checkLoginStatus() {
  const user = localStorage.getItem('currentUser');
  return user ? JSON.parse(user) : null;
}
```

#### 3.2.5 个人中心 — 侧滑面板与收藏/浏览历史

个人中心采用CSS transform驱动的侧滑面板（Slide-in Panel），支持收藏管理与浏览历史记录：

```javascript
// 收藏功能 — localStorage持久化
toggleFavorite(item) {
  const key = `favorites_${this.currentUser.username}`;
  let favorites = JSON.parse(localStorage.getItem(key) || '[]');
  const index = favorites.findIndex(f => f.id === item.id);
  if (index > -1) {
    favorites.splice(index, 1);  // 取消收藏
  } else {
    favorites.push({ ...item, addedAt: Date.now() });  // 添加收藏
  }
  localStorage.setItem(key, JSON.stringify(favorites));
},

// 浏览历史自动记录
addToHistory(place) {
  const key = `history_${this.currentUser.username}`;
  let history = JSON.parse(localStorage.getItem(key) || '[]');
  history = history.filter(h => h.id !== place.id);  // 去重
  history.unshift({ ...place, visitedAt: Date.now() });  // 最新在前
  history = history.slice(0, 50);  // 最多保留50条
  localStorage.setItem(key, JSON.stringify(history));
}
```

## 第六章 项目总结

### 6.1 项目成果

本项目成功构建了一个涵盖35+景点文化专题页面的青海安多藏族文化传播平台，实现了以下核心目标：

- **完整文化体系**：系统性地呈现了青海安多（11景点）、甘南安多（7景点）、川西阿坝（3景点）、藏北那曲（7景点）四大区域的文化风貌，是目前互联网上安多藏区覆盖最全面的文化展示平台
- **自研设计系统**：构建了基于CSS Custom Properties的统一Design System，包含色彩体系、三级阴影系统、四级响应式断点，确保35+页面的视觉一致性
- **交互式地图导航**：集成高德地图JS API，实现四区域折叠列表与地图标记的双向联动，提供沉浸式地理空间探索体验
- **完善的用户系统**：基于localStorage实现注册/登录/收藏/浏览历史/个人中心侧滑面板等个性化功能
- **丰富的藏文工具**：提供藏汉互译、藏文字符查询、常用短语学习、藏历查询、藏文键盘、藏文笔记等六大实用工具
- **良好的用户体验**：全站支持四级响应式布局，卡片式全区域点击交互，移动端触控友好

### 6.2 克服的困难

1. **藏汉翻译接口选型**：经历了从百度翻译API（需后端签名）→ 腾讯云TMT（存在CORS限制）→ MyMemory API（免费、免后端、浏览器直连）的多轮技术选型，最终选择了对纯前端架构最友好的方案
2. **地图服务选型与适配**：从OpenStreetMap切换至高德地图JS API，解决了中国区域地名显示、坐标系对齐、信息窗口中文排版等问题，确保符合国家测绘标准
3. **多区域内容编撰**：35+景点的文化内容需要兼顾学术准确性、通俗可读性和文化敏感性，涉及大量藏区地方志及学术文献调研
4. **藏文字符渲染兼容**：藏文Unicode字符在不同浏览器中的渲染表现差异较大，通过Google Fonts（Noto Sans Tibetan）和系统字体回退策略解决
5. **纯前端用户系统**：在无后端服务器的纯前端架构下，通过localStorage JSON序列化方案实现了完整的用户认证与数据持久化
