## 一、整体技术架构

### 1.1 技术栈概览

| 层级 | 技术选型 | 说明 |
|------|---------|------|
| 前端框架 | Vue 3 (CDN, Options API) | 响应式数据绑定、状态管理 |
| 样式系统 | CSS Custom Properties | 自研设计系统，CSS 变量驱动 |
| 地图服务 | 高德地图 JS API v1.4.15 | 景点标注、信息窗口、区域导航 |
| 翻译服务 | MyMemory Translation API | 藏汉双向即时翻译 |
| 数据持久化 | Web Storage API (localStorage) | 用户数据、笔记、收藏、浏览历史 |
| 图标 | Font Awesome 4.7.0 | 全站图标系统 |
| 字体 | Noto Sans SC (Google Fonts) | 中文无衬线体 |
| 藏历 | zangli-master (开源) | 1951 - 2050 年藏历数据 |

### 1.2 设计系统

| `--text-primary` | `#2D2D2D` | 正文深色 |
| `--text-secondary` | `#666666` | 辅助文字 |

#### 阴影与圆角

- 三级阴影系统：`--shadow-sm`（卡片静态）/ `--shadow-md`（悬停态）/ `--shadow-lg`（弹窗、浮层）

- 过渡时长：`0.3s ease`

## 二、地图模块

### 2.1 功能概述

集成高德地图 JS API，实现安多地区 **35 个景点** 的可视化展示，支持四区域折叠式导航、标记点-列表联动交互。

### 2.2 区域划分

| 区域 | 景点数量 | 代表景点 |
|------|---------|---------|
| 青海安多核心区 | 11 | 塔尔寺、青海湖、三江源、阿尼玛沁雪山 |
| 甘南安多文化区 | 8 | 拉卜楞寺、桑科草原、玛曲黄河第一弯 |
| 川西安多片区 | 8 | 阿坝州、若尔盖草原、红原大草原 |
| 藏北安多延伸区 | 8 | 那曲、纳木错、可可西里、唐古拉山口 |

### 2.3 技术实现

#### 地图初始化

```javascript
// 动态加载高德地图脚本
initAMap() → createMap()
// 关键参数
容器: #amap-container
中心点: [101.5, 35.5]
缩放级别: 7
视图模式: 3D
```

#### 右侧导航交互

- **折叠式区域列表**：四大区域默认折叠，点击区域标题展开/收起景点列表
- **均分空间布局**：四个区域块等高排列，使用 Flexbox `flex: 1` 均分右侧面板空间
- **边线间隔设计**：区域之间以细分隔线视觉区分，间距统一 `8px`

#### 标记点-列表联动

1. 点击右侧地名列表项 → 地图自动平移定位到对应景点 → 打开信息窗口 → 列表项高亮
2. 点击地图红色标记点 → 打开信息窗口（含景点图片、描述、"详细内容"链接）→ 对应列表项高亮
3. 通过 `activePlaceId` 变量跟踪当前激活景点，实现双向联动

#### 信息窗口

- 内容：景点名称 + 缩略图 + 简要描述 + "详细内容"跳转按钮
- 技术：HTML 模板字符串构建，`AMap.InfoWindow` 实例渲染
- 交互：点击跳转按钮进入对应景点专题页面

### 2.4 响应式适配

- `> 1024px`：左侧地图 + 右侧列表（左右布局）
- `< 1024px`：地图与列表上下堆叠
- `< 768px`：列表收起为可展开面板

## 三、用户认证模块

### 3.1 方案说明

> **重要更新**：已从早期设计的"手机号 + 验证码 + JWT 服务端认证"方案改为 **localStorage 本地存储** 方案。当前版本无后端服务，所有用户数据均存储在浏览器本地。

### 3.2 当前实现

#### 注册流程

1. 用户点击右上角用户按钮，弹出登录/注册弹窗
2. 切换至注册标签页
3. 输入**用户名**和**密码**
4. 点击注册按钮，用户数据以 JSON 格式存入 `localStorage`
5. 注册成功后自动切换至登录标签页

#### 登录流程

1. 输入用户名和密码
2. 从 `localStorage` 读取已注册用户数据，本地比对验证
3. 验证通过后写入 `authToken`（模拟令牌）和 `userName` 至 `localStorage`
4. 关闭弹窗，右上角显示用户登录状态

#### 状态管理

| localStorage Key | 数据类型 | 说明 |
|-----------------|---------|------|
| `authToken` | String | 登录状态标识（模拟令牌） |
| `userName` | String | 当前登录用户名 |
| `user_favorites` | JSON Array | 用户收藏列表（景点/民俗 ID 数组） |
| `browse_history` | JSON Array | 浏览历史记录（最多 50 条，FIFO） |

#### 个人中心

- **入口**：右上角用户头像下拉菜单 → 侧滑面板
- **功能**：收藏统计展示、浏览足迹时间线、一键清空历史

#### 功能访问控制

- 用户点击平台功能按钮时，系统检查 `authToken` 是否存在
- 未登录：自动弹出登录/注册弹窗
- 已登录：正常执行功能操作

### 3.3 表单验证规则

| 字段 | 规则 |
|------|------|
| 用户名 | 非空，至少 2 个字符 |
| 密码 | 6 - 20 位字符 |

### 3.4 后续扩展方向

后续可平滑扩展为服务端认证方案：

```
Spring Boot + MySQL + JWT
├── /api/user/register   POST  用户注册
├── /api/user/login      POST  用户登录（返回 JWT Token）
├── /api/user/logout     POST  用户登出
└── /api/user/info       GET   获取用户信息（Bearer Token 鉴权）
```

前端代码已预留 `fetch` 调用结构，接入后端后仅需将本地比对逻辑切换为 API 调用即可。

## 四、内容展示模块

### 4.1 模块概述

首页包含两大内容展示区域：**安多民俗文化** 和 **藏地旅游景点**，均采用横向滚动卡片轮播形式。

### 4.2 数据来源

- **当前方案**：本地 JavaScript 数据对象（非 API 驱动），数据硬编码在 Vue 实例的 `data` 中
- **降级机制**：代码中保留了 `fetch('/api/folk/list')` 和 `fetch('/api/tour-spot/list')` 的调用结构，API 请求失败时自动回退至默认本地数据

### 4.3 民俗文化卡片轮播

- **模块 ID**：`culture`
- **数据结构**：`{ id, title, desc, img }`
- **展示内容**：藏族情歌、赛马节、唐卡艺术、锅庄舞、婚俗、酥油花等
- **交互**：点击"了解更多"跳转至 `minsu.html?id=` 详情页

### 4.4 旅游景点卡片轮播

- **模块 ID**：`tour`
- **数据结构**：`{ id, title, desc, img }`
- **展示内容**：塔尔寺、青海湖、拉卜楞寺、三江源等
- **交互**：点击"查看详情"跳转至 `jingdian.html?id=` 详情页

### 4.5 卡片交互设计

| 交互行为 | 实现方式 |
|---------|---------|
| 悬停放大 | `transform: scale(1.03)` + `box-shadow` 增强 |
| 全区域点击 | 整张卡片可点击，非仅按钮 |
| 横向拖拽滚动 | `mousedown/mousemove/mouseup` 事件 + `scrollLeft` 计算 |
| 触摸滑动 | `touchstart/touchmove/touchend` 事件适配移动端 |
| 箭头导航 | 左右箭头按钮，每次滚动一个卡片宽度 |
| 分页指示器 | 底部圆点指示当前位置 |

## 五、藏文工具模块

### 5.1 模块总览

| 工具 | 入口文件 | 核心技术 |
|------|---------|---------|
| 藏汉互译 | `fanyi/fanyi.html` | MyMemory Translation API + AbortSignal 超时控制 |
| 藏文笔记 | `biji/biji.html` | localStorage JSON 数组 + 侧边栏列表管理 |
| 藏文键盘 | `jianpan/jianpan.html` | 虚拟键盘布局 + 字符映射 |
| 藏文字符查询 | `zifu/zifu.html` | 30 个基础藏文字符展示 |
| 常用短语 | `duanyu/duanyu.html` | 安多藏语分类短语展示 |
| 藏历查询 | `zangli-master/calendar.html` | zangli-master 开源库，1951 - 2050 年 |
| 藏文经典 | `jingdian/jingdian.html` | 《二十一度母礼赞》展开/收起翻译 + 夜间模式 |
| 藏族音乐 | `music/music.html` | 传统乐器/音乐展示 |

### 5.2 藏汉互译

- **API**：`https://api.mymemory.translated.net/get?q={text}&langpair={from}|{to}`
- **语言对**：`zh-CN ↔ bo`（中文 ↔ 藏文）
- **超时控制**：`AbortSignal.timeout(10000)`，超时后提示用户重试
- **复制功能**：翻译结果一键复制至剪贴板

### 5.3 藏文笔记

- **存储方式**：`localStorage` 中以 JSON 数组形式存储，Key 为 `tibetan_notes`
- **多条管理**：侧边栏列表展示所有笔记标题，点击切换编辑
- **功能**：新建、编辑、删除、时间戳记录
- **数据结构**：`{ id, title, content, createTime, updateTime }`

### 5.4 藏历查询

- **数据源**：zangli-master 开源项目，内含 1951 - 2050 年完整藏历数据
- **功能**：公历-藏历转换、节日查询、殊胜日标注
- **展示**：日历网格视图，支持年/月切换

## 七、性能与兼容性

### 7.1 性能优化策略

| 策略 | 实现方式 |
|------|---------|
| 资源延迟加载 | 高德地图 API 动态 `<script>` 注入，按需加载 |
| 图片优化 | 合理尺寸裁剪，减少初始加载体积 |
| DOM 操作优化 | Vue 虚拟 DOM 差量更新，减少重绘 |
| 事件委托 | 列表项点击事件统一委托至父容器 |
| CSS 动画 | 使用 `transform` 和 `opacity` 触发 GPU 加速 |

### 7.2 浏览器兼容性

- Chrome 80+、Firefox 78+、Safari 14+、Edge 80+
- 移动端：iOS Safari 14+、Android Chrome 80+

### 7.3 降级处理

- 高德地图加载失败 → 显示友好占位提示（图标 + 文字）
- 翻译 API 超时 → AbortSignal 10s 超时提示重试
- API 数据加载失败 → 自动回退本地内置数据
